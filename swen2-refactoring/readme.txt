Refactorn Sie den hier angegebenen Code. Achten Sie dabei insbesondere darauf, dass

- Methoden und Variablen aussagekräftige Namen haben

- Methoden möglichst wenige Parameter besitzen

- Methoden lediglich eine Abstraktionsebene abdecken und übersichtlich lang sind

- unterschiedliche Funktionalitäten in unterschiedlichen Klassen gekapselt werden

- kein duplizierter Code vorhanden ist

- möglichst nur die klasseneigenen Attriute zur Berechnung verwendet werden

- möglichst keinei case..switch-Anweisungen verwendet werden

- ... alles, was Sie sonst über Refactoring und Clean-Code gehört haben, berücksichtigt wird.

Prüfen Sie nach jeder Änderung, ob der Unittest weiterhin erfolgreich ausgeführt wird. Einige Änderungen können auch Änderungen am Unit-Test zurfolge haben.

P.S.: Falls Sie sich für die Mathematik hinter der GPS-Distanz-Berechnung interssieren, gibt es verschiedene Webseiten, die das Problem schön beschreiben.
